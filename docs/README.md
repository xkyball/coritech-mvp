# CoriTech Documentation

This directory contains CoriTech Phase 1 tickets, source-control ownership
records, and technical due-diligence placeholders.

## Contents

- `architecture/` - Architecture overview and build-vs-buy rationale.
- `data-model/` - Conceptual core data model and proof event model.
- `deployment/` - Deployment posture, environment configuration, and
  environment placeholders.
- `integrations/` - External systems register and vendor diligence fields.
- `product/` - Phase 1 MVP scope and delayed scope guardrails.
- `security/` - Security/GDPR note and role/permission starter matrix.
- `vendor-ip/` - IP, vendor and account ownership posture, including the
  account ownership checklist and IP assignment handover checklist.
- `tickets/phase-1/` - Codex-ready implementation tickets for Phase 1.
- `source-control/` - Repository ownership, branch protection, and review
  workflow evidence.

Documentation should stay ticket-driven. Placeholders are expected where account
identities, external platforms, or vendors have not been confirmed.

## Phase 1 Proof Frame

```text
Trigger -> Documentation -> Signature -> Verification Level -> Audit Trail
```

Delayed items are documented in `product/mvp-scope.md` and reflected across the
architecture, data, security, integration and deployment notes.
